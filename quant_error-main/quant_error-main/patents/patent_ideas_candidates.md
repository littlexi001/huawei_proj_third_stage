# 专利候选 idea（第一批）

本文档汇总 5 个与当前项目（谱感知量化 / Metis：低秩 head + residual，混合精度与时延约束）相关的软硬结合专利候选方向，供后续筛选与细化。

## Idea 1：面向 Metis 的“四项 GEMM + 残差合并”单算子融合执行（Metis-Fused GEMM Primitive）

- **瓶颈**：Metis 推理需要 4 次 GEMM（`W_hX_h / W_hX_r / W_rX_h / W_rX_r`）与结果融合；软件层多 kernel/多中间张量会放大 launch、HBM 往返与调度开销。
- **方案**：以执行描述符的形式输入低秩因子与 residual（低比特）表示，硬件在一个算子内完成四项乘法与累加融合；低秩项使用专用 micro-kernel/投影路径。
- **硬件参与点**：混合精度 GEMM 调度器 + tensor core 数据路径 + 融合累加单元。
- **可模拟指标**：kernel 启动次数、中间写回流量、端到端时延；精度不变（执行形态变化）。

## Idea 2：量化误差信号驱动的片上自适应 k/格式切换控制（Error-Signal Driven Precision Controller）

- **瓶颈**：最优 `k_a/k_p` 与 residual 格式会随层/输入变化；纯软件闭环自适应引入同步与策略开销。
- **方案**：在量化/反量化与 GEMM 路径中收集轻量统计（饱和/裁剪次数、零值比例、残差范数估计、近似谱能量比、周期计数等），控制器在时延预算约束下动态选择 `k_a/k_p` 与 residual 格式（FP4/FP8/BF16 回退），写入下一次执行描述符。
- **硬件参与点**：量化单元统计计数器 + 精度选择寄存器 + GEMM 调度器消费描述符。
- **可模拟指标**：固定时延预算下的精度提升；或固定精度下的平均 `k` 降低与时延下降。

## Idea 3：激活在线 SVD 的“基向量缓存 + 相似度门控复用”（Activation Subspace Cache）

- **瓶颈**：在线 SVD 代价高；batch/token 间子空间相似但复用逻辑若落在软件，容易引入额外 kernel 与数据搬运。
- **方案**：片上缓存低秩子空间参数（例如 `U_k/S_k/V_k` 或等价投影参数）；对新 token 先用缓存投影得到 `X_h/X_r`，再用轻量门控（抽样余弦相似度/残差能量阈值）判断是否重算一次近似 SVD 并更新缓存。
- **硬件参与点**：片上 SRAM cache + 投影/残差计算单元 + 门控统计器（输出复用/重算决策位）。
- **可模拟指标**：在线 SVD 次数、门控失败率 vs 精度损失曲线、端到端时延。

## Idea 4：token 语义簇驱动的分层量化策略与执行路由（Semantic-Aware Precision Routing）

- **瓶颈**：不同 token/语义对量化误差敏感性不同；对整个 batch 用同一配置会导致过度保精度或局部崩坏。
- **方案**：执行描述符携带 token 的簇 ID/敏感性档位（由 runtime/编译器或历史统计生成）；调度器按簇为 token 选择不同 `k_a` 或 residual 格式/回退策略，并优先让同簇 token 复用同一子空间缓存。
- **硬件参与点**：执行描述符元数据 + GEMM 调度器的分组分发。
- **可模拟指标**：固定预算下的长尾 token 精度提升、平均时延变化与簇内复用收益。

## Idea 5：FP4 residual 的“离群触发稀疏高精度纠偏通路”（Outlier-Triggered Correction Path）

- **瓶颈**：低比特误差常由少量饱和/裁剪元素主导；整块 residual 回退 BF16 代价过高。
- **方案**：量化阶段生成离群元数据（饱和位置/裁剪掩码或块级离群计数）；GEMM 时仅对被标记块/元素走小型 BF16 纠偏乘加旁路（稀疏增量项），其余仍走 FP4 主通路，并在输出侧融合。
- **硬件参与点**：量化单元元数据生成 + 稀疏纠偏单元 + 融合累加。
- **可模拟指标**：相同精度目标下的额外 FLOPs/时延开销；相同开销下相对纯 FP4 的精度提升。

## Idea 6：双尺度（Dual-Scale）FP4 块量化以抑制“小值被裁剪为 0”

- **瓶颈**：block-wise 量化的 scale 常被最大值主导，导致小值分辨率过粗并被系统性裁剪为 0。
- **方案**：对同一 block 同时维护两套 scale（或“主 scale + 次 scale + 阈值”）：大值用主 scale，小值用次 scale，从而在不改变主数据路径的前提下显著降低小值裁剪。
- **硬件参与点**：量化/反量化单元支持双尺度编码与解码；GEMM 数据路径按阈值选择对应 scale。
- **可模拟指标**：零值比例、裁剪/饱和计数、重构误差与下游精度；额外元数据开销与时延影响。

## Idea 7：低秩 head 的因子化存储格式与专用搬运/解码数据流（Head-Factor Packing & DMA）

- **瓶颈**：低秩 head 若在执行前被重建为稠密矩阵，会引入临时张量与额外访存/搬运，削弱低秩优势。
- **方案**：head 永远以因子形式（例如 `U/S/V`）持久化存储在 HBM，并采用专用布局与 DMA/解码器，将因子直接流式喂入投影或 micro-kernel；避免 head 稠密化与中间写回。
- **硬件参与点**：DMA/数据搬运引擎 + 低秩元数据解码器 + 寄存器/片上 SRAM 布局。
- **可模拟指标**：HBM 访问量、带宽占用、head 路径时延；与“稠密化再 GEMM”方案对比。

## Idea 8：无需 SVD 的硬件友好近似子空间提取（Sketch/Transform-Based Head Extraction）

- **瓶颈**：在线近似 SVD 仍然是 Metis 激活侧的主要时延来源之一。
- **方案**：使用硬件友好的随机/结构化变换与 sketch（例如块 Hadamard、随机投影、CountSketch 等）近似提取高能子空间，直接生成 head/residual 或等价的投影参数，在可控精度损失下替代在线 SVD。
- **硬件参与点**：轻量变换单元（可复用 FFT/Hadamard 类结构）+ 归约统计器 + 变换种子/参数寄存器。
- **可模拟指标**：在线分解开销、近似误差与下游精度；与近似 SVD 的时延-精度曲线对比。

## Idea 9：量化前谱能量预条件/旋转以压缩动态范围（Spectral Preconditioning for Quantization）

- **瓶颈**：参数/激活的谱各向异性导致数值动态范围过宽，使 block-wise 量化的 scale 被少数方向主导。
- **方案**：在进入低比特量化前，对矩阵块施加可逆的轻量正交变换（例如块内旋转/Householder/Hadamard 族），将能量从少数方向扩散，从源头降低极端最大值对 scale 的支配；变换可在计算后逆变换，或被吸收到相邻算子中。
- **硬件参与点**：量化单元前/后的小型变换网络 + 变换参数寄存器/描述符。
- **可模拟指标**：动态范围、饱和/裁剪率、重构误差；以及变换带来的额外 FLOPs/时延与净收益。

## Idea 10：端到端时延硬约束守门员：跨层动态降级与预算回收（Latency Guardrail）

- **瓶颈**：`k`、混合精度与在线分解的开销叠加后，端到端时延可能随输入分布波动，难以保证 SLA。
- **方案**：硬件维护每层真实周期/拥塞统计与剩余时延预算；当前层或早期层超支时，后续层自动触发降级策略（更小的 `k` 档位、更激进的低精度、关闭部分补偿路径），以保证端到端 SLA，并在预算富余时回收精度。
- **硬件参与点**：全局周期计数器 + per-layer budget 寄存器 + 调度器的降级策略表。
- **可模拟指标**：端到端时延分布与 SLA 违约率；在相同 SLA 下的平均精度与平均时延。
