# 专利：一种控制反馈本地化的迭代类算法GPU加速结构

## 摘要
本公开提出一种用于在并行处理单元内执行数据依赖型迭代计算的系统和方法。在一个实施例中，每个流式多处理器（stream multiprocessor，SM）包括一个本地控制处理器，该本地控制处理器被配置为执行依赖于向量执行单元所生成的中间计算结果的控制流。该控制处理器动态确定后续计算步骤，并分发相应操作，而无需主机处理器介入。通过将控制决策局部化在 SM 内，并将标量控制流与向量并行执行解耦，所述系统能够高效执行具有运行时依赖分支和收敛条件的迭代算法。所公开的架构减少了主机与设备之间的同步开销，改善了数据局部性，并提高了执行资源利用率。

## 背景技术
并行处理架构，尤其是图形处理器（graphics processing unit，GPU），通常被设计为通过单指令多线程（single-instruction multiple-thread，SIMT）等执行模型来利用数据级并行性。在此类系统中，通常称为 warp 的线程组在 warp 调度器的控制下以锁步方式执行指令。warp 调度器负责从就绪 warp 中进行选择，并向可用执行流水线发射指令，从而最大化吞吐量并隐藏延迟。

尽管该模型对于规则的、面向吞吐量的工作负载非常有效，但当其应用于包含数据依赖控制流的算法时，会表现出根本性局限。具体而言，许多具有实际重要性的数值算法，包括奇异值分解（SVD）、特征值分解、迭代求解器以及优化过程，都包含迭代流程，其中每个后续计算步骤依赖于执行过程中生成的中间结果。这些依赖关系通常表现为收敛检查、自适应分支、迭代参数的动态调整或提前终止条件。

在传统 GPU 执行模型中，此类控制流通常由主机处理器（例如中央处理器 CPU）管理。主机处理器在 GPU 上启动计算 kernel，取回中间结果，评估条件，并判断是否需要进一步计算。主机与设备之间的这种重复交互会引入显著开销，包括 kernel 启动延迟、同步延迟和数据传输成本。当每次迭代的计算工作量较小时，这些开销尤为突出，而小型或中型矩阵以及批处理操作中常常会出现这种情况。

使用持久化 kernel 或设备侧控制逻辑来缓解上述局限的尝试也会遇到额外挑战。在这些方案中，控制流使用 SIMT 执行资源实现，因而会由于线程之间的分支发散、标量操作带来的额外指令开销，以及线程或 warp 之间频繁同步的需求而造成低效利用。因此，现有架构范式缺乏一种能够将细粒度、数据依赖控制逻辑与向量并行计算高效结合执行的机制。

## 发明内容

本发明引入一种架构机制，使得数据依赖型迭代算法能够完全在流式多处理器（SM）内部执行，从而消除迭代控制循环期间对主机介入的需求。本发明基于在每个 SM 内引入本地控制处理器，该控制处理器被配置为执行依赖于向量执行单元所产生的中间结果的标量控制逻辑。

在一个实施例中，本地控制处理器维护与正在进行的计算相关联的控制状态，包括迭代计数器、收敛阈值和中间标量值等变量。控制处理器向 SM 内的执行单元分发向量操作，并接收标量摘要或条件指示形式的反馈。基于该反馈，控制处理器确定后续计算步骤，包括是否继续迭代、修改参数或终止执行。

该架构有效地将控制流执行与向量并行计算解耦。向量执行单元保持专用于高吞吐量算术操作，而控制处理器执行迭代算法所需的顺序型、数据依赖逻辑。通过将控制和数据都局部化在 SM 内，系统实现了更低延迟、更好的数据局部性以及更高的执行效率。

## 具体实施方式

### 总体系统架构

在一个实施例中，处理系统包括多个流式多处理器（SM），每个 SM 均能够执行并行工作负载。每个 SM 包括：一组被配置为对向量或矩阵执行算术操作的向量执行单元；被配置为向执行单元发射指令的 warp 调度器；用于存储中间数据的寄存器文件和共享内存；以及被配置为执行控制逻辑的本地控制处理器。

向量执行单元可以包括算术逻辑单元（ALU）、浮点单元、张量处理单元，或用于执行并行计算的其他专用硬件。warp 调度器以传统方式运行，根据执行资源可用性和依赖约束从就绪 warp 中进行选择并发射指令。

本地控制处理器集成在 SM 内，并与现有组件并行运行。不同于从已有指令流中进行选择的 warp 调度器，控制处理器基于运行时条件动态生成并编排计算步骤。

### 本地控制处理器架构

在一个实施例中，本地控制处理器包括一个标量处理核心，该标量处理核心包括标量算术逻辑单元（ALU）、控制寄存器文件、分支与比较逻辑，以及控制指令存储器或微码存储器。控制处理器被配置为执行一系列控制指令，这些控制指令实现迭代算法的逻辑。

控制指令存储器可以存储一个微程序，该微程序编码特定算法的控制流，包括循环结构、条件分支和终止条件。控制处理器以类似传统标量处理器的方式取指并执行这些指令，但其指令集可以是针对控制操作定制的受限指令集。

控制处理器维护控制状态，该控制状态包括迭代索引、收敛阈值和中间标量结果等变量。这些变量存储在控制寄存器文件中，并随着计算推进而更新。

### 控制与计算交互

本地控制处理器通过命令接口与向量执行单元交互。在一个实施例中，控制处理器发出命令，该命令指定由向量执行单元执行的操作。这些命令可以包括向量加法、点积、范数计算、矩阵-向量乘法或其他线性代数原语。

每个命令包括足以识别操作、输入操作数和输出位置的信息。输入操作数可以位于 SM 的寄存器文件或共享内存中，输出结果写回这些存储位置。

在一个向量操作完成后，执行单元生成一个或多个标量摘要值，例如范数、残差或收敛度量。这些标量值通过反馈路径返回给控制处理器。

### 反馈与决策机制

控制处理器接收来自向量执行单元的标量反馈，并使用该信息评估决定计算下一步骤的条件。例如，控制处理器可以将残差值与收敛阈值进行比较，以判断迭代过程是否已经收敛。

基于该评估结果，控制处理器可以采取多个动作之一。其可以分发另一个向量操作以继续迭代，调整步长或活动集大小等参数，或者在收敛条件满足时终止计算。

这种由反馈驱动的决策过程使控制处理器能够在不涉及主机处理器的情况下实现复杂的数据依赖控制流。

### 执行模型

所公开系统的执行模型与传统 GPU 执行模型存在根本差异。在所公开系统中，迭代算法的控制循环完全在 SM 内执行。控制处理器以闭环方式初始化计算、分发向量操作、接收反馈、评估条件并确定后续动作。

这种闭环执行消除了迭代过程期间对主机-设备同步的需求。主机处理器可以通过提供输入数据和控制参数来发起计算，但不参与迭代控制循环。

### 示例：奇异值分解

在一个实施例中，所公开系统用于对矩阵执行奇异值分解（SVD）。控制处理器通过将矩阵加载到共享内存并设置初始参数来初始化 SVD 计算。

在每次迭代期间，控制处理器向向量执行单元分发向量操作，以计算奇异向量、残差或中间变换等量。执行单元执行这些计算，并返回指示收敛进展的标量值。

控制处理器评估这些值，以判断奇异值是否已经在指定容差范围内收敛。如果尚未收敛，控制处理器确定下一迭代步骤并分发相应操作。该过程持续进行直至达到收敛，此时控制处理器终止计算。

### 支持的计算类型与泛化计算模式

在一些实施例中，本公开不限于奇异值分解（SVD），而是适用于更广泛的一类计算。该类计算具有如下特征：在流式多处理器（SM）内进行分阶段并行计算，生成中间反馈，进行局部条件评估，并选择后续计算步骤。在这些实施例中，向量执行资源可以执行批量算术操作，本地控制处理器可以评估由这些操作生成的一个或多个中间标量摘要，并且可以在无需主机处理器重复介入的情况下确定一个或多个后续计算步骤。

在一些实施例中，所公开架构可以应用于分解类计算。此类计算可以包括但不限于奇异值分解、特征值分解、QR 分解、幂迭代、Lanczos 迭代、子空间迭代、主成分提取、低秩近似以及低秩更新过程。在这些实施例中，向量执行单元可以执行矩阵-向量乘法、点积计算、范数计算、正交化相关操作或其他线性代数原语，而本地控制处理器维护包括例如迭代次数、收敛阈值、秩参数、残差度量或其他标量控制参数在内的控制状态。基于一个或多个中间标量摘要，例如残差值、范数或收敛指示，本地控制处理器可以确定是否继续执行、终止执行、修改一个或多个控制参数，或者转入不同的计算阶段。

在一些实施例中，所公开架构可以应用于由归约驱动的计算和归一化类计算。此类计算可以包括但不限于 softmax、层归一化、RMS 归一化、均值计算、方差计算、平方和计算、梯度范数计算以及梯度裁剪。在这些实施例中，可以首先执行一个或多个归约操作以获得一个或多个标量统计量，然后可以基于这些统计量选择一个或多个后续的向量范围或张量范围操作，或者对其进行参数化。举例而言，softmax 计算可以包括最大值归约阶段、指数运算阶段、求和归约阶段和归一化阶段。类似地，归一化计算可以包括在应用归一化操作之前获得均值、方差或 RMS 统计量。在这些实施例中，本地控制处理器可以接收一个或多个中间标量摘要，包括例如最大值、求和结果、均值、方差、RMS 值或范数，并可基于这些摘要确定一个或多个后续向量操作的顺序、条件和/或参数化方式。

在一些实施例中，所公开架构可以应用于自适应控制类计算和决策驱动型执行流。此类计算可以包括但不限于动态损失缩放、溢出检测、下溢检测、量化尺度更新、裁剪阈值调整、活动集大小调整、步长调整、混合专家路由、top-$k$ 选择、稀疏激活选择、token 选择以及条件执行路径选择。在这些实施例中，向量执行资源可以在一组数据元素上计算分数、统计量或候选度量，而本地控制处理器可以评估所得反馈并确定一个或多个后续动作，包括例如选择路由、启用或禁用分支、调整阈值、修改尺度，或终止某个执行阶段。

因此，在一些实施例中，本公开适用于一种泛化计算模式，其中向量执行资源执行并行算术运算，该算术运算生成一个或多个中间标量摘要，本地控制处理器评估这些摘要，本地控制处理器确定一个或多个后续计算步骤、参数更新或终止条件，并重复上述动作，直到满足指定的完成条件。因此，所公开架构不限于 SVD 专用实现，而是扩展到更广泛的一类反馈驱动、数据依赖且局部闭环的计算。

### 本地控制处理器与 warp 调度器之间的关系

在一些实施例中，本地控制处理器可以与 warp 调度器共存于同一流式多处理器内。warp 调度器可以保留其传统功能，即在 SIMT 执行模型下选择并发射与就绪 warp 相关联的指令。相比之下，本地控制处理器可以管理选定反馈驱动计算的局部闭环执行，包括维护控制状态、接收一个或多个中间标量摘要、评估一个或多个条件，并确定一个或多个后续向量操作。

在一些实施例中，warp 调度器和本地控制处理器运行于不同控制域之上。warp 调度器可以控制已有线程级指令流在可用执行资源上的发射，例如用于维持传统 GPU 工作负载的吞吐量并隐藏延迟。相比之下，本地控制处理器可以基于 SM 内生成的运行时反馈，控制数据依赖任务的后续计算步骤选择。因此，warp 调度器可以管理已有指令流如何被发射，而本地控制处理器可以管理针对选定任务类别接下来应执行什么操作或阶段。

在一些实施例中，warp 调度器和本地控制处理器可以共享同一 SM 内的一个或多个向量执行资源、寄存器文件和/或共享内存资源，同时控制不同的执行语义。例如，warp 调度器可以代表传统 SIMT warp 发射操作，而本地控制处理器可以代表局部闭环任务实例发射操作。二者之间的协调可以使用例如保留规则、优先级规则、基于分区的分配、时间复用或其他仲裁机制来实现。在一些实施例中，可以为由本地控制处理器管理的任务保留一部分执行带宽、寄存器空间或共享内存空间。在一些实施例中，SM 可以被划分为多个本地域，每个本地域与对应的本地控制处理器相关联。

在一些实施例中，本地控制处理器提供传统 warp 调度器并不自然具备的功能。warp 调度器通常被配置为从已有线程上下文中发射指令，而不是评估中间标量反馈、更新本地算法状态，并为迭代型、归约驱动型或其他反馈驱动型任务确定一个或多个后续计算步骤。如果将此类控制功能映射到 SIMT 模型下的普通线程上，可能导致线程发散、额外标量指令开销、重复同步以及控制逻辑执行低效。因此，在一些实施例中，本地控制处理器为选定任务类别提供了独立的本地控制平面，而不是重复 warp 调度器的功能。

### 示例：奇异值分解的示性执行流程

在一些实施例中，主机处理器可以向选定流式多处理器或一组流式多处理器提交任务描述符。该任务描述符可以包括但不限于算法标识符、一个或多个输入数据位置、一个或多个输出数据位置、一个或多个初始控制参数、收敛阈值、最大迭代次数，以及控制程序、微码序列或状态机的入口点。在这些实施例中，主机处理器可以发起执行并提供初始配置信息，但不需要参与随后的本地迭代控制循环。

在一些实施例中，在接收到任务描述符之后，本地控制处理器可以执行本地初始化过程。该初始化可以包括将一个或多个输入矩阵、向量或张量加载到寄存器文件和/或共享内存中，在控制寄存器文件中建立初始控制状态，设置一个或多个标量状态变量，例如迭代计数器、阈值、活动集大小、步长、秩参数或尺度参数，并加载与所提交任务相关联的一个或多个控制指令、微码条目或状态机设置。

在一些实施例中，初始化之后，本地控制处理器可以通过命令接口向一个或多个向量执行单元发出第一组向量命令。这些向量命令可以指定但不限于操作类型、一个或多个输入操作数位置、一个或多个输出位置、目标寄存器地址、目标共享内存地址和/或一个或多个模式位或控制参数。所发出的操作可以包括但不限于矩阵-向量乘法、点积计算、范数计算、归约求和计算、最大值计算、最小值计算、归一化准备计算、候选分数计算以及其他线性代数或归约原语。

在一些实施例中，在当前向量阶段完成之后，一个或多个向量执行单元可以将一个或多个结果写入寄存器文件和/或共享内存，并且还可以通过反馈路径向本地控制处理器提供一个或多个中间标量摘要。此类标量摘要可以包括但不限于范数、残差、收敛度量、最大值、最小值、求和结果、均值、方差、溢出指示、裁剪指示、路由统计量或 top-$k$ 摘要。

在一些实施例中，本地控制处理器可以基于返回的标量摘要评估一个或多个条件，并相应更新本地控制状态。作为示例而非限制，本地控制处理器可以将残差与收敛阈值进行比较，基于计算出的范数确定是否执行裁剪，基于求和结果、最大值或方差确定是否调用后续归约阶段，调整步长、活动集大小、秩参数或量化尺度，递增迭代计数器，和/或设置一个或多个终止标志或异常标志。

在一些实施例中，如果终止条件未被满足，本地控制处理器可以确定一个或多个后续向量操作，并为下一计算阶段发出一个或多个相应命令。该下一阶段可以对应但不限于当前迭代内的后续子步骤、下一次迭代的第一步、参数调整后的重试阶段，或根据运行时条件选择的分支依赖阶段。以这种方式，所公开架构在 SM 内形成包括向量执行、标量反馈生成、条件评估以及后续命令发射的局部闭环。

在一些实施例中，当完成条件被满足时，例如达到收敛、耗尽最大迭代次数或出现其他终止准则时，本地控制处理器可以终止本地执行循环并执行一个或多个完成动作。此类完成动作可以包括但不限于置位完成信号、将一个或多个最终结果写入指定存储位置、写入状态字或摘要统计量、释放本地控制资源以供后续任务使用，和/或通知主机处理器或更高层级调度单元执行已完成。

因此，在一些实施例中，所公开的方法可以包括：从主机处理器提交任务和一个或多个初始参数；由本地控制处理器在 SM 内建立本地控制状态；使用向量执行资源执行第一阶段并行计算；生成并返回一个或多个中间标量摘要；评估一个或多个条件并更新本地状态；确定并发出一个或多个后续向量操作；重复上述一个或多个动作，直到满足终止条件；以及输出一个或多个最终结果。该方法使反馈驱动任务的控制循环能够保持在 SM 本地，从而可以减少主机-设备往返，减少与细粒度控制逻辑相关的 SIMT 发散，并改善数据局部性和执行响应性。

### 替代实施例

本地控制处理器可以以多种形式实现。在一些实施例中，其可以是具有受限指令集的可编程微控制器。在其他实施例中，其可以是执行预定义操作序列的微码序列器。在另一些实施例中，其可以实现为有限状态机（FSM），该有限状态机基于输入条件在不同状态之间转换。

控制处理器的数量和布置可以变化。在一些实施例中，每个 SM 包括单个控制处理器。在其他实施例中，SM 的每个分区包括单独的控制处理器，以允许更细粒度的控制。
